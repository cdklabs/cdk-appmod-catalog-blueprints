  <action>
Create execute_script.py tool that validates scripts and invokes execution Lambda.

The tool must:

1. **Import ast_validator:**
   ```python
   from ast_validator import validate_script
   ```

2. **Define @tool decorated function:**
   ```python
   @tool
   def execute_script(script: str, row_count: int = 100) -> Dict[str, Any]:
       """
       Validate and execute a Python DataGenerator script.

       Args:
           script: Python code containing DataGenerator class
           row_count: Number of preview rows to generate (max 100 for preview)

       Returns:
           Dictionary with schema, preview data, or error details
       """
   ```

3. **Validation with self-healing loop:**
   ```python
   MAX_FIX_ATTEMPTS = 3
   current_script = script

   for attempt in range(MAX_FIX_ATTEMPTS + 1):
       validation = validate_script(current_script)

       if validation['valid']:
           break

       if attempt == MAX_FIX_ATTEMPTS:
           return {
               'success': False,
               'error': 'Script validation failed after 3 fix attempts. Please rephrase your request.',
               'validation_errors': validation['errors'],
               'recoverable': True
           }

       # Self-healing: invoke BatchAgent to fix the script
       fix_result = invoke_batch_agent_for_fix(
           current_script,
           validation['errors']
       )
       if not fix_result['success']:
           return fix_result
       current_script = fix_result['script']
   ```

4. **invoke_batch_agent_for_fix() helper function:**
   - Get BATCH_AGENT_FUNCTION_NAME from environment
   - Build payload with script and specific errors to fix
   - Invoke Lambda and parse response
   - Return fixed script or error

5. **After validation passes, invoke execution Lambda:**
   - Get EXECUTION_LAMBDA_NAME from environment (will be wired in Plan 02-02)
   - Build payload with validated script and row_count
   - Handle response: parse schema and preview data
   - Return structured result:
     ```python
     {
         'success': True,
         'schema': {...},  # Column definitions
         'preview': [...],  # First 100 rows
         'row_count': int,
         'summary': 'Generated 6-column dataset preview with 100 rows'
     }
     ```

6. **Error handling:**
   - Catch Lambda invocation errors
   - Return recoverable=True for transient errors
   - Return recoverable=False for configuration errors

7. **Use same patterns as generate_script.py:**
   - sanitize_input() is NOT needed here (script comes from BatchAgent)
   - Follow same response format with success/error/recoverable

Note: EXECUTION_LAMBDA_NAME environment variable will be wired in Plan 02-02.
For now, if the env var is missing, return a clear error that execution is not yet configured.
